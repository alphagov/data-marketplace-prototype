Data Catalogue Metadata API (3.1.1)

Download OpenAPI specification:Download

URL: https://developer.datamarketplace.gov.uk

This API allows you to search, create, update and delete metadata records about data assets published in any data catalogue that implements the data-catalogue-metadata JSON schema.

This JSON schema is based on the Data Catalog Vocabulary (DCAT) - Version 3.

An example of a data catalogue implementing this schema is the CDDO Data Marketplace.

Cataloged Resources

Operations concerning the cataloged-resource model.

queryDataAssets

Performs a filtered query against your organisations data assets.

Authorizations:
ClientCredentialsScheme
query Parameters
filter
string <= 30 characters
Example: filter=demographics

Returns data asset metadata where a match is found against the following fields:

  • title
  • keywords
  • supplier identifier

Datasets

Operations concerning the dataset model.

CreateDataSet

Creates a dataset resource. The path to the newly created resource will be returned in the Location response header.

Authorizations:
ClientCredentialsScheme
Request Body schema: application/json
accessRights
string
Enum: "INTERNAL" "OPEN" "COMMERCIAL"

A rights statement that concerns how the distribution is accessed.

required
Array of objects

Contact information

description
required
string <= 4096 characters

A concise narrative of the content of a cataloged-resource.

identifier
string <= 50 characters

A unique number, code, or reference value assigned to a cataloged-resource within the catalogue.

keyword
Array of strings[ items <= 75 characters ]

Uncontrolled terms (words or phrases) assigned to describe an information resource.

YYYY-MM-DD (string) or string (dateOrDateTime)
publisher
string <= 75 characters

The entity responsible for making the cataloged-resource available in the catalogue.

securityClassification
string
Enum: "OFFICIAL" "SECRET" "TOP_SECRET" "NOT_APPLICABLE"

An information security designation that identifies the minimum level of protection assigned to an information resource.

status
string
Enum: "Draft" "Published" "Withdrawn" "Deleted"

Lifecycle status of a cataloged-resource within the catalogue.

supplierIdentifier
string <= 50 characters

A unique number, code, or reference value assigned to a cataloged-resource in a third party (supplier's) system.

theme
Array of strings
Items Enum: "Agriculture, fisheries and forestry" "Business, economics and finance" "Crime and justice" "Culture, leisure and sport" "Education" "Energy" "Environment and nature" "Geography" "Government and public sector" "Health and care" "Population and society" "Science and technology" "Transport and infrastructure"

Topic: A controlled term that expresses the broad topical content of an information resource. Subject: A controlled term that expresses a topic of the intellectual content of an information resource.

title
required
string <= 200 characters

The full and formal name given to an information resource. A name given to the distribution.

type
required
string
Enum: "Data Set" "Data Service" "Data Group" "Data Share"

Topic: A controlled term that expresses the broad topical content of an information resource. Subject: A controlled term that expresses a topic of the intellectual content of an information resource.

Array of objects

A collection of distributions

YYYY-MM-DD (string) or string (dateOrDateTime)
updateFrequency
string

The schedule for how often the data is updated
