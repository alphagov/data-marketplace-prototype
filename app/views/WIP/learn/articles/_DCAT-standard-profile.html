Adding a collection of metadata records as a CSV file

You can add data descriptions using a CSV file. This can be used to provide multiple data descriptions in a single process. The descriptions must be added to the provided CSV file template, which has been structured to ensure descriptions conform to the cross-government data sharing metadata model.

Step 1: Gather information about your data description

Use the questions as a guide to ensure enough information can be provided at each stage of adding a data description

You need to answer the questions to make sure your published data description will be helpful to users trying to find data. You should avoid using domain specific jargon, acronyms and abbreviations. Write your description in plain English.

Step 2: Entering your descriptions into the CSV file

Download the CSV template file and provide your answers to the questions under the relevant columns.

The first tab in the spreadsheet provides some guidance on using the template. The Dataset tab should be completed with the values for your dataset descriptions (the data made available as a file download). The DataService tab should be completed with the values for your data service descriptions (the data that is made available as a web API).

Step 3: Uploading the CSV file to the Data Marketplace

Sign in to the Data Marketplace and navigate to the Publish data descriptions dashboard. Then click on the ‘Add new data description’ button. On the next page you will need to select 'Upload a CSV file of data asset descriptions'.



You will now be taken through to a page where you can download the CSV template file for providing your data descriptions. When you are ready, click on the Upload CSV file button.



On the next page, you can upload the CSV file containing your data descriptions. You will need to select the CSV file from the device you want to upload.

Step 4: Review your entered descriptions

Once you have uploaded your file, you will see an Upload summary page, where you can check the status of your uploaded file.

For a file with an ERROR, note the reason and amend the CSV file before selecting the 'Re-upload data' button. You will be taken through the upload process again.



If no errors are found, you will see a summary page of the data descriptions that have been extracted from the CSV file. You should review each entry to ensure the values have been captured correctly. If you find something that doesn’t look right, you should fix this in your CSV file and select the Re-upload CSV file option.

Once you are happy that the data descriptions are correct, you can select the Publish data asset description button which will add the data descriptions to the Data Marketplace. Finally, you will be shown a success page and given the option to return to the publisher dashboard.
