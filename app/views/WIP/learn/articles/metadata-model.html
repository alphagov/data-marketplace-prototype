Metadata sharing requirements

This specification focuses on the metadata attributes needed to describe essential shared data assets in the context of a cross-government data share and to facilitate data discoverability in the Government Data Marketplace. For a high-level overview see the guidance page for using metadata to describe data assets in a data catalogue.


Key terms

The definitions for the key terms are taken from the Data Catalog Vocabulary DCAT published by the World Wide Web Consortium. The Cross-Government Metadata Exchange Model is a profile defined over DCAT.

Term Definition
Catalogue A curated collection of metadata about resources.
Data Resource Resource published or curated by a single agent.
Data Service A collection of operations that provides access to one or more datasets or data processing functions.
Dataset A collection of data, published or curated by a single agent, and available for access or download in one or more representations.
Dataset Series A collection of datasets that are published separately, but share some characteristics that group them.
Distribution A specific representation of a dataset. A dataset might be available in multiple serialisations that may differ in various ways, including natural language, media-type or format, schematic organisation, temporal and spatial resolution, level of detail or profiles (which might specify any or all of the above).
