Publishing data descriptions

The Data Marketplace allows government organisations to publish descriptions of data (known as metadata) so they can be found by anyone in government. Once found, a data share request can be made to facilitate access to the data under the agreed terms. The Data Marketplace does not host the data itself or provide access to the data. Your data description will be published to the Data Marketplace and will be available to be found by other government organisations who are looking for data.

All data descriptions should be findable, accessible, interoperable and reusable (FAIR). With data being published to the Data Marketplace, it should be easy to find, labelled with correct access rights, described to support interoperability with other data, and well described to support reuse. The descriptions of data should have their own identifiers and remain in the marketplace even if the data is no longer available.

The Data Marketplace supports descriptions of datasets and data services. A dataset is a collection of data, published or curated by a single agent and available for access or download in one or more representations. A data service is a collection of operations that provides access to one or more datasets or data processing functions.

Data descriptions can be added in a number of ways including:

  • Manually through a web form
  • Upload via our CSV template
  • Catalogue connection to synchronise the descriptions in your organisation data catalogue with the Data Marketplace (using a data harvester)

To publish data descriptions you must have a Data Marketplace account with publish permission for your organisation. See X for how to get an account with publish permissions. The publisher dashboard will give you an overview of the data your organisation has published in the Data Marketplace and how it is being used.

Creating a manual data description

Add data descriptions manually

You can add a data description to the Data Marketplace using an online form. You must submit one form per dataset or data service.

Step 1: Gather information about your data description

Use the questions as a guide to ensure that enough information can be provided at each stage of adding a data description.

By answering the questions, your published data description will be helpful to users trying to find data. You should avoid using domain specific jargon and acronyms. Write your description in plain English.

The questions also ensure that your description meets the cross-government data sharing metadata standards.

Step 2: Enter your data description

Sign into the Data Marketplace and navigate to the Publish data descriptions dashboard then click on the ‘Add new data description’ button.



You need to select whether you are adding the description of a dataset (something that is distributed as a file download such as a CSV) or a data service (a web API that provides access to the data). If you are describing a dataset, then select ‘Add dataset description manually’. If you are describing a data service then select ‘Add a data service description manually’.



You will now be taken through a series of forms where you can provide the answers to each of the questions.

Step 3: Review your description

Once you have completed entering the description of your data, you will be presented with a Check your answers page where you will have the option to change any of your answers.



Once you are satisfied your answers are correct, click on Submit, which will confirm your description of the data has been uploaded.

Step 4: View published data descriptions

Your data description will now have been added to the Data Marketplace. If you click on View metadata record, you will be taken to the page that provides the details of your data. To return to the home page, click on ‘Home’.
